# API Server NodeJS

<Head>
    <link rel="canonical" href="https://docs.appseed.us/boilerplate-code/api-server/node-js/" />
</Head>

<SubHeading>Open-source Node JS API Server enhanced with JWT Authentication and SQLite storage.</SubHeading>

Free API Server coded on top of Express / Node JS with **SQLite** persistence and JWT authentication via Passport library - `passport-jwt` strategy.

:::info [**v1.0.6**](https://github.com/app-generator/api-server-nodejs/releases) - release date `Dec 2, 2022`
:::

- [API Definition](api-unified-definition.mdx) - the unified API structure implemented by this server
- [Node JS API Server](https://github.com/app-generator/api-server-nodejs) - source code
- Full-stack samples compatible with this product:
  - [React Berry Dashboard](https://github.com/app-generator/react-berry-admin-template) - open-source sample
  - [React Datta Dashboard](https://github.com/app-generator/react-datta-able-dashboard) - open-source sample

<br />

> **API Interface** - for full description please access [API Unified Definition](api-unified-definition.mdx)

- USERS API:
  - `/api/users/register`: create a new user
  - `/api/users/login`: authenticate an existing user
  - `/api/users/logout`: delete the associated JWT token
  - `/api/users/checkSession`: check an existing JWT Token for validity
  - `/api/users/edit` - edit the information associated with a registered user

## How to use the code

> ðŸ‘‰ **Step 1** - Clone the sources

```
$ git clone https://github.com/app-generator/api-server-nodejs.git
$ cd api-server-nodejs
```

> ðŸ‘‰ **Step 2** - Install dependencies via NPM or Yarn

```
$ npm i
// OR
$ yarn
```

> ðŸ‘‰ **Step 3** - Run the `SQLite` migration

```
$ yarn typeorm migration:run
```

> ðŸ‘‰ **Step 4** - Start the API server (development mode)

```
$ npm dev
// OR
$ yarn dev
```

> ðŸ‘‰ **Step 5** - Production Build (files generated in `build` directory)

```
$ npm build
// OR
$ yarn build
```

> ðŸ‘‰ **Step 6** - Start the API server (production mode)

```
$ npm start
// OR
$ yarn start
```

The API server will start using the `PORT` specified in `.env` file (default 5000)


## Codebase Structure

```
< ROOT / src >
     |
     |-- config/
     |    |-- config.ts             # Configuration
     |    |-- passport.ts           # Define Passport Strategy
     |
     |-- migration/
     |    |-- some_migration.ts     # database migrations
     |
     |-- models/
     |    |-- activeSession.ts      # Sessions Model (Typeorm)
     |    |-- user.ts               # User Model (Typeorm)
     |
     |-- routes/
     |    |-- users.ts              # Define Users API Routes
     |
     |
     |-- index.js                     # API Entry Point
     |-- .env                       # Specify the ENV variables
     |
     |-- ************************************************************************
```
